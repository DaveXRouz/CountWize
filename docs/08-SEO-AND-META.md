## Structured Data (JSON-LD)

### FAQPage Schema

Implemented on FAQ pages via JavaScript:

```javascript
const faqData = [];
document.querySelectorAll('.faq-item').forEach(item => {
  const question = item.querySelector('.faq-item-header').textContent;
  const answer = item.querySelector('.faq-item-answer').textContent;
  faqData.push({
    "@type": "Question",
    "name": question,
    "acceptedAnswer": {
      "@type": "Answer",
      "text": answer
    }
  });
});

const jsonLd = {
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": faqData
};

const script = document.createElement('script');
script.type = 'application/ld+json';
script.textContent = JSON.stringify(jsonLd);
document.head.appendChild(script);
```

### Breadcrumb Schema

Implemented on detail pages:

```html
<nav aria-label="Breadcrumb">

    <li itemprop="itemListElement" itemscope itemtype="https://schema.org/ListItem">
      <a itemprop="item" href="index.html">
        <span itemprop="name">Home</span>
      </a>
      <meta itemprop="position" content="1">
    </li>
    <!-- Additional items -->
  </ol>
</nav>
```

## URL Best Practices

### Clean URLs

Implemented via Netlify redirects:
- `/recovery` instead of `/recovery.html`
- `/about-us` instead of `/about-us.html`

### Canonical URLs

Every page has a canonical URL:
```html
<link href="https://www.countwize.com/[page]" rel="canonical">
```

### 301 Redirects

Duplicate content consolidated:
- `/contact.html` → `/contact-us.html`
- `/crypto-education` → `/education.html`
- `/faq-crypto-recovery` → `/faq.html`
