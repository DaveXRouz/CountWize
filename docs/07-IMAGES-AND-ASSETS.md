## Naming Conventions

### 1. Responsive Variants

Pattern: `{name}-p-{width}.{ext}` or `{name}-{size}x{size}q{quality}.{ext}`

| Suffix | Width | Purpose |
|--------|-------|---------|
| `-p-130x130q80` | 130px | Thumbnails (80% quality) |
| `-p-500` | 500px | Mobile |
| `-p-800` | 800px | Tablet |
| `-p-1080` | 1080px | Small desktop |
| `-p-1600` | 1600px | Desktop |
| `-p-2000` | 2000px | High resolution |
| `-p-2600` | 2600px | Ultra HD |
| `-p-3200` | 3200px | Ultra HD+ |

Example:
```
1.jpg
1-p-500.jpg
1-p-800.jpg
1-p-1080.jpg
1-p-1600.jpg
1-p-2000.jpg
```

### 2. Descriptive Names

Pattern: `{number}-{UPPERCASE-TITLE}.{ext}`

Example: `1-INTRODUCTION-TO-CRYPTOCURRENCY-AND-SECURITY.png`

### 3. Hash-Based Names (CMS)

Pattern: `{32-character-hash}_1.webp`

Example: `7e69734df533cc85922e762d462c41c779d476b7_1.webp`

### 4. Timestamp Names

Pattern: `{timestamp}.webp`

Example: `1746119745842.webp`

## Image Usage in HTML

### Responsive Images (srcset)

```html
<img
  src="images/1.jpg"
  loading="lazy"
  sizes="(max-width: 479px) 94vw, (max-width: 767px) 95vw, (max-width: 991px) 92vw, (max-width: 1439px) 734px, (max-width: 1919px) 51vw, 734px"
  srcset="
    images/1-p-500.jpg 500w,
    images/1-p-800.jpg 800w,
    images/1-p-1080.jpg 1080w,
    images/1-p-1600.jpg 1600w,
    images/1-p-2000.jpg 2000w,
    images/1-p-2600.jpg 2600w,
    images/1-p-3200.jpg 3200w,
    images/1.jpg 3504w"
  alt="Description"
>
```

### Lazy Loading

All non-critical images use:
```html
<img loading="lazy" ...>
```

Critical/hero images use:
```html
<img loading="eager" ...>
```

### GPU Acceleration

```css
img {
  will-change: transform;
}
```

## Image Optimization Notes

1. **WebP Format** - Modern browsers get optimized WebP versions
2. **Multiple Sizes** - Responsive variants for all breakpoints
3. **Lazy Loading** - Non-critical images load on demand
4. **SVG Icons** - Vector graphics for scalability
5. **Compression** - Quality balanced for file size
6. **CDN Ready** - Netlify serves with proper caching headers
